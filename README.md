# Rule Mirror

A Vue 3 + Vite static site that mirrors multiple rule repositories, designed for deployment on Tencent Cloud EdgeOne Pages.

## Mirrored Projects

- [MetaCubeX/meta-rules-dat](https://github.com/MetaCubeX/meta-rules-dat) - GeoIP / GeoSite Database
- [Loyalsoldier/clash-rules](https://github.com/Loyalsoldier/clash-rules) - Rule Sets (RULE-SET)
- [mnixry/direct-android-ruleset](https://github.com/mnixry/direct-android-ruleset) - Android CN Apps & Games Direct Rule Sets

## How It Works

1. **GitHub Action**: Runs daily (or manually) to sync files
   - Downloads latest release assets from multiple repositories
   - Organizes files by project in separate directories
   - Generates a JSON manifest with project information
   - Commits the files to the `main` branch

2. **Frontend**: Displays files grouped by project
   - Fetches `/files.json` to get the project and file list
   - Shows projects in separate sections
   - Displays file names, sizes, checksums, and download links
   - Supports expanding checksums for verification

3. **Deployment**: Tencent EdgeOne Pages automatically builds and deploys
   - Monitors the `main` branch for changes
   - Runs `pnpm build` to generate static files
   - Deploys the built site

## Local Development

```bash
# Install pnpm globally (if not installed)
npm install -g pnpm

# Install dependencies
pnpm install

# Start dev server
pnpm dev

# Build for production
pnpm build

# Preview production build
pnpm preview
```

## Deployment to Tencent Cloud EdgeOne Pages

1. Fork this repository
2. Enable GitHub Actions in your fork
3. Connect your repository to Tencent Cloud EdgeOne Pages
4. Configure build settings:
   - Build command: `pnpm build`
   - Output directory: `dist`
5. The GitHub Action will automatically sync files daily

## Manual Sync

Go to the "Actions" tab in your repository and run the "Sync Rule Files" workflow manually.
